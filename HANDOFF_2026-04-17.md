## Important query format note

The UI currently still uses the older "LERF" naming in places, but for the Gaussian backend it now routes into the new SigLIP-based query engine.

The query box is **not** free-form natural language. It expects semicolon-separated tokens:

### For `grasp` and `twist`

- `object;part`
- example: `flower;petal`
- example: `cup;rim`

### For `pick & place` and `pour`

- `object;part;place`
- example: `flower;petal;table`

If the number of semicolon-separated fields is wrong, the UI will reject the query before it even reaches the semantic backend.

## Why the query error happens

In `scripts/gen_grasp.py`, the query text is parsed using:

```python
lerf_word = gen_grasp_text.value.split(";")
```

Then it hard-checks the count:

- `grasp` / `twist` must have exactly 2 entries
- `pick & place` / `pour` must have exactly 3 entries

So these will fail:

- `flower`
- `pick up flower`
- `flower petal`

And these will pass the format check:

- `flower;petal`
- `cup;rim`
- `flower;petal;table`
